# Phase 2, Plan 02: Homepage Sections

**Phase Goal**: A visitor landing on the homepage understands what SMEAI offers within 5 seconds, learns about Ohm's background, and can reach a dedicated contact page
**Plan Scope**: Hero section with value proposition, About section with vet-to-AI story and portfolio link, Services overview, CTA section, and rewrite of index.astro to compose them all
**Requirements**: LAND-01 (value proposition in 5 seconds), LAND-02 (about section with vet-to-AI story), LAND-04 (portfolio link visible)
**Depends on**: 02-01 (contact.ts constants, contact page exists for hero CTA link)

## Context

Plan 01 created `src/lib/contact.ts` and the `/contact` page. This plan builds the homepage by creating four section components in `src/components/landing/` and rewriting `src/pages/index.astro` to compose them.

Key codebase patterns to follow:
- Container: `max-w-5xl mx-auto px-4` (matching Header nav container, NOT `prose-container`)
- Colors: `surface-*` for backgrounds/text, `primary-*` for CTAs/links, `accent-*` for highlights
- Dark mode: every color class needs a `dark:` counterpart
- Icons: Heroicons outline SVGs inlined directly (no library)
- All components are `.astro` files (zero client JS for static sections)
- Contact data imported from `src/lib/contact.ts`

Photo handling: Ohm's actual photo is not yet in the repo. The About section will use a placeholder div with initials. The code will include a commented-out `<Image />` import ready to swap in when the photo is available.
